Небольшой учебный проект для проверки проектного плана.

Приложение позволяет загрузить список задач, показать его в виде диаграммы Ганта и в 3d-визуализации и найти простые ошибки в плане.

## Что умеет

- загружать проект из JSON;
- загружать таблицу Excel или CSV;
- добавлять задачи вручную;
- показывать диаграмму Ганта;
- показывать загрузку исполнителей;
- показывать 3D-представление задач;
- находить ошибки в плане;
- экспортировать простой текстовый отчёт.

## Как открыть

1. Скачайте архив проекта.
2. Распакуйте его.
3. Откройте файл `index.html` в браузере.

Для диаграммы Ганта, Excel и 3D-вида нужен интернет, потому что часть библиотек подключается через CDN.

## Формат Excel / CSV

В таблице должны быть такие колонки:

| Колонка | Что писать |
|---|---|
| id | номер задачи |
| text | название задачи |
| start_date | дата начала |
| end_date | дата окончания |
| assignee | ответственный |
| category | категория |
| deliverable | результат задачи |
| depends_on | от каких задач зависит |

Даты лучше писать в формате:

2026-07-01

Если задача зависит от нескольких задач, можно написать так:

1,2


## Формат JSON

Пример:

```json:
{
  "meta": {
    "name": "Название проекта",
    "description": "Описание проекта"
  },
  "tasks": [
    {
      "id": 1,
      "text": "Анализ требований",
      "start_date": "2026-07-01",
      "end_date": "2026-07-03",
      "assignee": "Иван",
      "category": "Аналитика",
      "deliverable": "Список требований"
    }
  ],
  "links": [
    {
      "id": 1,
      "source": 1,
      "target": 2,
      "type": "finish_to_start"
    }
  ]
}
```

## Какие ошибки проверяются

Инструмент проверяет:

- есть ли название задачи;
- указаны ли даты;
- правильно ли указаны даты;
- есть ли ответственный;
- есть ли результат задачи;
- не заканчивается ли задача раньше начала;
- нет ли слишком длинных задач;
- нет ли неправильных зависимостей;
- нет ли циклов в зависимостях;
- не назначен ли один человек на несколько задач одновременно;
- есть ли финальная задача.
